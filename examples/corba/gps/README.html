PocoCapsule/C++ example: CORBA GPS

This example is originated from an OMG's tutorial designed for CORBA Component Model (CCM). It models a GPS instrument consists of three components implemented as Plain Old CORBA local objects:

  • a tick pulse generator local CORBA object, implemented as TickGenImpl, generates clock pulses for the instrument,

  • a GPS location device local CORBA object, implemented as GPSLocatorImpl, locates the instruments current location and notifies subscribed displays on position update.

  • a display device local CORBA object, implemented as NavDisplayImpl, retrieve location data from GPS and display them.


These three components (local CORBA objects) are instantiated and wired up using the PocoCapsule/C++ IoC Container according to the setup.xml description.

Source Files

main.C: This is a simple container used to assemble the GPS application context.

Interfaces.idl: IDL CORBA local interfaces definitions of all component CORBA objects. Note: An equivalent POCO version of this example, that uses C++ pure virtual classes to define component object interfaces, can be found in the ${POCOCAPSULE_DIR}/examples/basic-ioc/gps directory.

TickGenImpl.h and TickGenImpl.C: The implementation of tick generator device CORBA local object.

GPSLocatorImpl.h and GPSLocatorImpl.C: The implementation of GPS locator device CORBA local object.

NavDisplayImpl.h and NavDisplayImpl.C: The implementation of Navigate Display device CORBA local object. It is also notable that above three local CORBA object implementation themselves neither contain code, nor link with PocoCapsule/C++ container runtime engine. They are plain old CORBA local objects that can be developed and tested completely independent of and unaware of PocoCapsule/C++ container.

setup.xml: The application context descriptor that describes the configuration and wiring of the modeled GPS application.

Building this example

To build this example, the environment variable POCOCAPSULE_DIR should point to the PocoCapsule/C++ installed directory. Also, this example assumes an underneath ORB (e.g. VisiBroker/C++, TAO, etc.) is installed and its runtime and development environment (such as POCOCAPSULE_DIR, VBROKER_DIR or TAO_ROOT, etc. env variable) are set according to its product specification. Then, this example can be built by simply invoking gmake/make on linux/unix or nmake on windows.

Running this example

· Before starting the application deployment container (i.e. the main executable), make sure the LD_LIBRARY_PATH (on linux and solaris) or the PATH (on windows) environment variable is set correctly to include the ${POCOCAPSULE_DIR}/lib directory and the ${VBROKREDIR}/lib (if VisiBroker is used) or the ${TAO_ROOT}/lib directory (if TAO is used).

· run this example by execute the main executable: 

prompt> main

It will display GPS locations ...
